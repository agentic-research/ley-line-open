## Dependency-tree observations

`cargo tree --duplicates --manifest-path rs/Cargo.toml` shows 100+ duplicate-version pairs across the workspace. The overwhelming majority are transitive duplications inherent to the crates.io graph (multiple major versions of `rand`, `getrandom`, `bitflags`, etc., reachable through different dep chains). Not actionable.

**Direct-dep duplication eliminated by this PR:** `nalgebra v0.32 vs v0.35` was being created by the partial bump (`sheaf` on 0.35 but `nalgebra-sparse 0.9` still pulling 0.32). Bumping `nalgebra-sparse` to 0.12 collapses this to a single 0.35 version.

**Direct-dep duplications NOT eliminated:**

- `thiserror v1 vs v2` ÔÇö was an inconsistency between `sign` (1.0) and `sheaf` / `text-search` (2). PR unifies on `2`. Workspace is now uniform.
- `rusqlite v0.39` ÔÇö already uniform; no action.
